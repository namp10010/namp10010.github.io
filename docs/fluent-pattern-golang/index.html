Golang fluent pattern

Posted on 2020, Oct 24 by Nam Pham 2 mins read

Simple implementation of fluent pattern in Go.

Constructor style

The gist of fluent pattern is returning the same object over and over again.

type email struct {
  from    string
  to      string
  subject string
  body    string
}

func (e *email) withFrom(fr string) *email {
  e.from = fr
  return e
}

func (e *email) withTo(to string) *email {
  e.to = to
  return e
}

func (e *email) withSubject(s string) *email {
  e.subject = s
  return e
}

func (e *email) withBody(b string) *email {
  e.body = b
  return e
}

Now, the object can be created by

  myEmail := &email{}
  myEmail.withFrom("sender").
    withTo("recipient").
    withSubject("hello").
    withBody("howdy?")

Run the code on go playground. See source code

Middleware style

As functions in Go are also object, one can also create fluent pattern for functions

type emailAction func(payload string)

func (a emailAction) connectToServer(server string) emailAction {
    return func(payload string) {
        fmt.Printf("connecting to server: %v\n", server)
        a(payload)
    }
}

func (a emailAction) authenticate(cred string) emailAction {
    return func(payload string) {
        fmt.Printf("authenticating with cred: %v\n", cred)
        a(payload)
    }
}

This way functions can be chained together

    var sendEmail emailAction = func(payload string) {
        fmt.Printf("sending email: %v\n", payload)
    }

    action := sendEmail.
        authenticate("username:password").
        connectToServer("email-server:443")

    action(fmt.Sprint(*myEmail))

The order of constructing action may seem counter intuitive. However, it is indeed correct because it’s just a chain of function calls. My colleague Lei likes to call these function wraps onion layers.

action() ~ connectToServer()
    |
    |
    authenticate()
        |
        |
        sendEmail()

Running this code will show

connecting to server: email-server:443
authenticating with cred: username:password
sending email: {sender recipient hello howdy?}

Run on Go playground. See source code.

This pattern is quite popular in package net/http for the implementation of interceptor middlewares.
